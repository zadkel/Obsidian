## **리눅스 방화벽 (Firewall)**

### **1. 개요**

- **목적:** 네트워크 트래픽을 미리 정의된 보안 규칙에 따라 제어하여 시스템을 보호하는 핵심 보안 요소임.
- **기능:** 주로 패킷 필터링(IP 주소, 포트 번호, 프로토콜 기반)을 통해 작동함.

### **2. 주요 방화벽 도구

- 리눅스 배포판에 따라 주로 사용되는 방화벽 관리 도구가 다름.
- **`firewalld`:** RHEL 계열 기본 방화벽. **존(Zone)** 개념 사용, 동적 관리 지원. 관리 명령어 `firewall-cmd`.
- **`ufw` (Uncomplicated Firewall):** Debian/Ubuntu 계열 기본 방화벽. 사용 편의성 강조. 관리 명령어 `ufw`.
- **`iptables`:** 전통적인 커널 패킷 필터링 도구. 강력하지만 복잡함.

### **3. `firewalld` 사용법 (RHEL 계열)**

- **핵심 개념:** 존(Zone) 기반 규칙 관리, Runtime(임시) 설정과 Permanent(영구) 설정 구분. 영구 설정 적용 시 `--reload` 필요.

 
**주요 `firewall-cmd` 명령어:**

| 명령어                                                    | 핵심 설명                                               |
| :----------------------------------------------------- | :-------------------------------------------------- |
| `systemctl status firewalld`                           | `firewalld` 서비스 상태 확인                               |
| `systemctl start\|stop\|restart firewalld`             | `firewalld` 서비스 시작 / 중지 / 재시작                       |
| `firewall-cmd --state`                                 | `firewalld` 실행 상태 확인 (running / not running)        |
| `firewall-cmd --get-default-zone`                      | 기본 존(Zone) 이름 확인                                    |
| `firewall-cmd --get-active-zones`                      | 현재 활성화된 존과 할당된 네트워크 인터페이스 확인                        |
| `firewall-cmd --list-all`                              | 기본 존의 모든 설정 (인터페이스, 서비스, 포트 등) 확인                   |
| `firewall-cmd --list-all --zone=<zone_name>`           | 특정 존의 모든 설정 확인                                      |
| `firewall-cmd --list-services [--permanent]`           | 현재(또는 영구) 설정에 허용된 서비스 목록 확인                         |
| `firewall-cmd --list-ports [--permanent]`              | 현재(또는 영구) 설정에 허용된 포트 목록 확인                          |
| `firewall-cmd --add-service=<서비스명> [--permanent]`      | 특정 서비스 트래픽 허용 규칙 추가 (예: `http`, `ssh`)              |
| `firewall-cmd --remove-service=<서비스명> [--permanent]`   | 서비스 허용 규칙 제거                                        |
| `firewall-cmd --add-port=<포트>/<프로토콜> [--permanent]`    | 특정 포트/프로토콜 트래픽 허용 규칙 추가 (예: `8080/tcp`, `1194/udp`) |
| `firewall-cmd --remove-port=<포트>/<프로토콜> [--permanent]` | 포트 허용 규칙 제거                                         |
| `firewall-cmd --reload`                                | 영구(permanent) 설정을 현재 실행(runtime) 설정으로 다시 로드하여 적용    |

### **4. `ufw` 사용법 (Debian/Ubuntu 계열)**

- **핵심 개념:** 단순한 명령어 구조, 기본 정책(Default Policy) 설정 중요 (일반적으로 incoming deny, outgoing allow).

 **주요 `ufw` 명령어:**

| 명령어                                        | 핵심 설명                                           |
| :----------------------------------------- | :---------------------------------------------- |
| `ufw status`                               | 방화벽 활성화 상태 및 간단한 규칙 목록 확인                       |
| `ufw status verbose`                       | 자세한 규칙 목록 및 기본 정책 확인                            |
| `ufw status numbered`                      | 규칙 목록을 번호와 함께 확인 (규칙 삭제 시 유용)                   |
| `ufw enable`                               | 방화벽 활성화 (부팅 시 자동 시작 설정 포함)                      |
| `ufw disable`                              | 방화벽 비활성화                                        |
| `ufw default deny incoming`                | 기본적으로 들어오는 모든 트래픽 차단 정책 설정                      |
| `ufw default allow outgoing`               | 기본적으로 나가는 모든 트래픽 허용 정책 설정                       |
| `ufw allow <서비스명 \| 포트번호[/프로토콜]>`          | 특정 서비스(예: `ssh`) 또는 포트/프로토콜(예: `80/tcp`) 트래픽 허용 |
| `ufw deny <서비스명 \| 포트번호[/프로토콜]>`           | 특정 서비스 또는 포트/프로토콜 트래픽 차단                        |
| `ufw allow from <IP주소> [to any port <포트>]` | 특정 IP 주소로부터의 접근 허용 (특정 포트 지정 가능)                |
| `ufw delete <규칙번호 \| 규칙내용>`                | 번호 또는 규칙 내용을 지정하여 기존 규칙 삭제                      |
| `ufw reset`                                | 모든 규칙 초기화 및 방화벽 비활성화 (**주의!**)                  |

# 링크 
## 🔗 아웃링크
```dataview
LIST WITHOUT ID outgoing
FROM ""
WHERE file = this.file
FLATTEN file.outlinks as outgoing
SORT outgoing ASC
```
## 🔙 백링크
```dataview
LIST WITHOUT ID file.link
WHERE contains(file.outlinks, this.file.link)
SORT file.link ASC
```
