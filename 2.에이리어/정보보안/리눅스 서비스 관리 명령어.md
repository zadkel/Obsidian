## **리눅스 서비스 관리 명령어 (systemctl)**

`systemd` 시스템 및 서비스 매니저를 제어하는 기본 명령어. 백그라운드에서 실행되는 서비스(데몬)의 상태를 관리하고 제어하는 데 사용됨.

- **관리 대상:** `systemd`는 '유닛(Unit)' 단위로 관리함. 서비스는 주로 `.service` 확장자를 가진 유닛 파일로 정의됨. 명령어에서 `<unit>` 부분에는 보통 `<서비스명>.service` (확장자 생략 가능) 형식으로 서비스 이름을 지정함.

| 명령어                                         | 핵심 설명                                                     |
| :------------------------------------------ | :-------------------------------------------------------- |
| `systemctl start <unit>`                    | 지정한 서비스 유닛 시작                                             |
| `systemctl stop <unit>`                     | 지정한 서비스 유닛 중지                                             |
| `systemctl restart <unit>`                  | 지정한 서비스 유닛 재시작 (중지 후 시작)                                  |
| `systemctl reload <unit>`                   | 서비스 중단 없이 설정 파일 다시 로드 (일부 서비스만 지원)                        |
| `systemctl status <unit>`                   | 지정한 서비스 유닛의 현재 상태(활성 여부, 로그 등) 상세 확인                      |
| `systemctl enable <unit>`                   | 시스템 부팅 시 서비스 자동 시작되도록 활성화                                 |
| `systemctl disable <unit>`                  | 시스템 부팅 시 서비스 자동 시작되지 않도록 비활성화                             |
| `systemctl is-enabled <unit>`               | 서비스가 부팅 시 자동 시작되도록 설정되었는지 확인                              |
| `systemctl is-active <unit>`                | 서비스가 현재 실행 중(활성 상태)인지 확인                                  |
| `systemctl list-units --type=service`       | 현재 메모리에 로드되어 **실행 중**이거나 **실행했던** 서비스 유닛 목록 표시            |
| `systemctl list-units --type=service --all` | 비활성 상태 포함, 현재 로드된 모든 서비스 유닛 목록 표시                         |
| `systemctl list-unit-files --type=service`  | **설치된 모든** 서비스 유닛 파일과 자동 시작 설정(enabled/disabled) 상태 목록 표시 |
| `systemctl mask <unit>`                     | 서비스를 완전히 비활성화 (수동 시작 및 의존성으로 인한 시작 모두 차단, `disable`보다 강력) |
| `systemctl unmask <unit>`                   | `mask`로 비활성화된 서비스 해제                                      |

*참고: `<unit>` 부분에는 관리하려는 서비스의 유닛 이름(예: `httpd.service` 또는 `httpd`)을 입력*

# 링크 
## 🔗 아웃링크
```dataview
LIST WITHOUT ID outgoing
FROM ""
WHERE file = this.file
FLATTEN file.outlinks as outgoing
SORT outgoing ASC
```
## 🔙 백링크
```dataview
LIST WITHOUT ID file.link
WHERE contains(file.outlinks, this.file.link)
SORT file.link ASC
```
