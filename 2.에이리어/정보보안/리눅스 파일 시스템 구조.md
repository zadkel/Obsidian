## **리눅스 파일 시스템 구조 (FHS)**

리눅스 및 유닉스 계열 운영체제에서 사용되는 **파일 시스템 계층 구조 표준 (Filesystem Hierarchy Standard, FHS)**. 시스템 파일 및 디렉토리를 목적에 맞게 표준화된 위치에 구성하여 일관성과 관리 용이성을 높임.


| 디렉토리           | 핵심 용도                                            |
| -------------- | ------------------------------------------------ |
| `/`            | 파일 시스템의 최상위 시작점. 모든 디렉토리의 부모.                    |
| `/bin`         | 기본적인 실행 명령어 (모든 사용자가 사용 가능).                     |
| `/sbin`        | 시스템 관리용 실행 명령어 (주로 root 사용자가 사용).                |
| `/etc`         | 시스템 전체의 주요 설정 파일 저장 위치.                          |
| `/home`        | 일반 사용자들의 홈 디렉토리 생성 위치 (/home/사용자명).              |
| `/root`        | root 사용자의 홈 디렉토리.                                |
| `/lib, /lib64` | 시스템 및 프로그램 실행에 필요한 공유 라이브러리 파일 저장.               |
| `/var`         | 시스템 운영 중 수시로 변경되는 데이터 저장 (로그, 스풀, 캐시 등).         |
| `↳ /var/log`   | 시스템 및 각종 애플리케이션의 로그 파일 저장 위치.                    |
| `/tmp`         | 임시 파일 저장 공간 (재부팅 시 내용 삭제될 수 있음).                 |
| `/usr`         | 사용자가 설치한 프로그램, 라이브러리, 문서 등 저장.                   |
| `↳ /usr/bin`   | /bin 외의 일반 사용자용 명령어.                             |
| `↳ /usr/sbin`  | /sbin 외의 시스템 관리자용 명령어.                           |
| `↳ /usr/lib`   | /lib 외의 공유 라이브러리.                                |
| `↳ /usr/local` | 시스템 관리자가 로컬로 설치한 소프트웨어 저장 (배포판과 별도 관리).          |
| `/dev`         | 시스템 장치(Device) 파일 저장 위치 (하드디스크, 터미널 등).          |
| `/proc`        | 커널 및 실행 중인 프로세스 정보를 가상 파일 형태로 제공.                |
| `/boot`        | 시스템 부팅 관련 파일 저장 (커널 이미지, 부트로더 설정 등).             |
| `/opt`         | 추가적인(Optional) 서드파티 응용 소프트웨어 설치 위치.              |
| `/mnt, /media` | 외부 저장 장치(USB, CD-ROM 등)를 **일시적으로 연결(마운트)**하는 지점. |

# 링크 
## 🔗 아웃링크
```dataview
LIST WITHOUT ID outgoing
FROM ""
WHERE file = this.file
FLATTEN file.outlinks as outgoing
SORT outgoing ASC
```
## 🔙 백링크
```dataview
LIST WITHOUT ID file.link
WHERE contains(file.outlinks, this.file.link)
SORT file.link ASC
```
