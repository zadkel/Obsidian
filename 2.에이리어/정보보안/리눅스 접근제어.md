## **접근 제어 (Access Control)**

### **1. 정의 (Definition)**

- **핵심:** 어떤 주체(Subject, 예: 사용자, 프로세스)가 어떤 객체(Object, 예: 파일, 시스템 자원)에 대해 어떤 행동(Operation, 예: 읽기, 쓰기, 실행)을 할 수 있는지 **허가 여부를 결정하고 통제**하는 과정 또는 기술.
- **목표:** 정보 보안의 3요소(CIA Triad) 보장 지원.
    - **기밀성 (Confidentiality):** 허가되지 않은 사용자의 정보 접근 차단.
    - **무결성 (Integrity):** 허가되지 않은 사용자의 정보 변경 및 손상 차단.
    - **가용성 (Availability):** 허가된 사용자가 필요할 때 정보 및 자원에 접근할 수 있도록 보장.

### **2. 접근 제어 모델 (Access Control Models)**

- 접근 제어 정책을 구현하기 위한 **이론적 틀 또는 방식**을 의미함.
- 대표적인 모델로 Bell-LaPadula, Biba 등이 있으며, 주로 **강제적 접근 제어(MAC, Mandatory Access Control)** 환경에서 논의됨. MAC는 시스템 관리자가 설정한 규칙(보안 등급 등)에 따라 접근 통제가 강제되는 방식임.

### **3. Bell-LaPadula 모델 (벨-라파듈라 모델)**

- **주요 목표:** **기밀성 (Confidentiality)** 보장. 정보의 불법적인 유출 방지에 초점.
- **환경:** 주로 군사, 정부 기관 등 보안 등급이 중요한 환경에서 사용됨.
- **핵심 규칙:** 주체와 객체에 보안 등급(예: Top Secret > Secret > Confidential > Unclassified)을 부여하고 다음 규칙 적용.
    - **단순 보안 속성 (Simple Security Property / No Read Up - NRU):** 주체는 **자신보다 높은 보안 등급의 객체를 읽을 수 없음.** (인가되지 않은 정보 획득 방지)
    - **스타 속성 (*-Property / No Write Down - NWD):** 주체는 **자신보다 낮은 보안 등급의 객체에 쓸 수 없음.** (높은 등급의 정보가 낮은 등급으로 유출되는 것 방지)
- **요약:** "위로는 읽지 못하고, 아래로는 쓰지 못한다." (==No Read Up, No Write Down==)

### **4. Biba 모델 (비바 모델)**

- **주요 목표:** **무결성 (Integrity)** 보장. 정보의 불법적인 수정 및 오염 방지에 초점. Bell-LaPadula 모델의 기밀성 규칙과 반대(Dual)되는 개념 적용.
- **환경:** 금융 정보, 중요한 시스템 데이터 등 정보의 정확성과 신뢰성이 중요한 환경에서 사용됨.
- **핵심 규칙:** 주체와 객체에 무결성 등급(예: Critical > Important > Unknown)을 부여하고 다음 규칙 적용.
    - **단순 무결성 속성 (Simple Integrity Property / No Read Down - NRD):** 주체는 **자신보다 낮은 무결성 등급의 객체를 읽을 수 없음.** (낮은 신뢰도의 정보가 높은 신뢰도의 주체/프로세스를 오염시키는 것 방지)
    - **스타 무결성 속성 (*-Integrity Property / No Write Up - NWU):** 주체는 **자신보다 높은 무결성 등급의 객체에 쓸 수 없음.** (낮은 신뢰도의 주체가 높은 신뢰도의 정보를 수정/훼손하는 것 방지)
- **요약:** "아래로는 읽지 못하고, 위로는 쓰지 못한다." (==No Read Down , No Write Up==)

# 링크 
## 🔗 아웃링크
```dataview
LIST WITHOUT ID outgoing
FROM ""
WHERE file = this.file
FLATTEN file.outlinks as outgoing
SORT outgoing ASC
```
## 🔙 백링크
```dataview
LIST WITHOUT ID file.link
WHERE contains(file.outlinks, this.file.link)
SORT file.link ASC
```
