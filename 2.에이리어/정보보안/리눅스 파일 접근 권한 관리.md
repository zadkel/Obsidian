## **리눅스 파일 접근 권한 관리**

### **1. 권한의 이해 (Understanding Permissions)**

- 리눅스는 파일 및 디렉토리에 대한 접근을 통제하기 위해 **권한(Permission)** 메커니즘을 사용함.
- **접근 주체 (대상):**
    - **소유자 (User/Owner):** 파일을 생성한 사용자.
    - **그룹 (Group):** 파일 소유자가 속한 그룹 또는 파일에 지정된 그룹. 그룹에 속한 여러 사용자가 동일한 권한 공유 가능.
    - **기타 사용자 (Others):** 소유자도 아니고 그룹 멤버도 아닌 나머지 모든 사용자.
- **권한 종류:** 각 주체별로 세 가지 기본 권한 부여 가능.
    - **읽기 (Read - r):** 파일 내용 읽기, 디렉토리 내용(파일 목록) 보기 허용.
    - **쓰기 (Write - w):** 파일 내용 수정/삭제, 디렉토리 내 파일 생성/삭제/이름 변경 허용.
    - **실행 (Execute - x):** 파일을 프로그램처럼 실행 허용, 디렉토리 접근(해당 디렉토리로 `cd` 이동 등) 허용.

### **2. 권한 확인 (`ls -l` 명령어)**

- 터미널에서 `ls -l` 명령어를 사용하여 파일/디렉토리의 권한 및 소유자 정보를 확인할 수 있음.
- **출력 형식 예시:** `-rwxr-xr-- 1 user group 1024 Apr 14 11:20 file.txt`
    - **첫 번째 문자:** 파일 종류 (`-`: 일반 파일, `d`: 디렉토리, `l`: 심볼릭 링크 등)
    - **다음 9개 문자:** 권한 정보 (총 3개 섹션)
        - `rwx` (1-3번째): 소유자(user)의 읽기(r), 쓰기(w), 실행(x) 권한.
        - `r-x` (4-6번째): 그룹(group)의 읽기(r), 실행(x) 권한 (쓰기 권한 없음 `-`).
        - `r--` (7-9번째): 기타 사용자(others)의 읽기(r) 권한 (쓰기, 실행 권한 없음 `-`).
    - **이후 정보:** 링크 수, 소유자 이름, 그룹 이름, 파일 크기, 최종 수정 시간, 파일/디렉토리 이름 순서로 표시됨.

### **3. 권한 변경 (`chmod` 명령어)**

- `chmod` (Change Mode) 명령어는 **파일이나 디렉토리의 접근 권한을 변경**하는 데 사용됨. 두 가지 모드(심볼릭, 8진수)로 사용 가능.

#### **가. 심볼릭 모드 (Symbolic Mode)**

- 문자 기호를 조합하여 직관적으로 권한을 추가, 삭제, 설정하는 방식. 아래 표의 기호들을 조합하여 사용함.
- **기본 형식:** `chmod [대상][연산자][권한] 파일/디렉토리명`

| 구분                  | 기호  | 설명                         |
| :------------------ | :-- | :------------------------- |
| **사용자 (대상)**        | u   | 파일 또는 디렉토리 소유자 (User)      |
|                     | g   | 그룹 소유자 (Group)             |
|                     | o   | 기타 사용자 (Others)            |
|                     | a   | 모든 사용자 (All - u, g, o 모두)  |
|                     |     | (대상을 명시하지 않으면 기본값 `a`)     |
| **동작 명령 (연산)**      | +   | 권한 추가                      |
|                     | -   | 권한 삭제                      |
|                     | =   | 권한 지정 (지정된 권한만 설정, 나머진 제거) |
| **Permission (권한)** | r   | 읽기 권한                      |
|                     | w   | 쓰기 권한                      |
|                     | x   | 실행 권한                      |
**사용 예시:**

- `chmod u+x script.sh`: 소유자에게 실행 권한 추가.
- `chmod g-w data.txt`: 그룹의 쓰기 권한 제거.
- `chmod o=r config.conf`: 기타 사용자에게 읽기 권한만 부여 (다른 권한은 제거됨).
- `chmod a+r README`: 모든 사용자에게 읽기 권한 추가.
- `chmod ug+rw shared_dir`: 소유자와 그룹에게 읽기, 쓰기 권한 동시 추가.

#### **나. 8진수 모드 (Octal Mode)**

- 권한을 숫자로 표현하여 설정하는 방식. 각 권한은 다음 값을 가짐: `r = 4`, `w = 2`, `x = 1`.
- 각 대상(소유자, 그룹, 기타 사용자)별로 권한 값의 합계를 계산하여 세 자리 숫자로 표현함.
    - `rwx` =  `7`
    - `rw-` =  `6`
    - `r-x` =  `5`
    - `r--` =  `4`
    - `--x` =  `1`
    - `---` =  `0`
- **기본 형식:** `chmod [8진수_권한값] 파일/디렉토리명`
- **사용 예시:**
    - `chmod 755 script.sh`: user `rwx`(7), group `r-x`(5), others `r-x`(5) 권한 설정. (스크립트 파일에 흔히 사용)
    - `chmod 644 data.txt`: user `rw-`(6), group `r--`(4), others `r--`(4) 권한 설정. (일반 텍스트 파일에 흔히 사용)
    - `chmod 700 private_dir`: user `rwx`(7), group `---`(0), others `---`(0) 권한 설정. (소유자만 접근 가능한 디렉토리)

### **4. 소유권 변경 (`chown`, `chgrp` 명령어)**

- **파일이나 디렉토리의 소유자 또는 소유 그룹을 변경**해야 할 경우 사용.
- `chown` (Change Owner): 소유자 및 그룹을 변경함.
    - `chown <새_소유자> <파일/디렉토리>`: 소유자만 변경.
    - `chown <새_소유자>:<새_그룹> <파일/디렉토리>`: 소유자와 그룹 동시 변경.
    - `chown :<새_그룹> <파일/디렉토리>`: 그룹만 변경 (chgrp와 유사).
    - `-R` 옵션: 디렉토리와 그 안의 모든 하위 파일/디렉토리 소유권을 재귀적으로 변경.
- `chgrp` (Change Group): 소유 그룹만 변경함.
    - `chgrp <새_그룹> <파일/디렉토리>`
    - `-R` 옵션: 재귀적으로 그룹 변경.


_(참고: `chown`, `chgrp` 명령어는 주로 root 사용자 또는 현재 소유자만 사용할 수 있는 등 제약이 있을 수 있음)_

### **5. 기본 권한 설정 (`umask`)**

- `umask` (User file-creation mode mask)는 사용자가 **새로운 파일이나 디렉토리를 생성할 때 부여되는 기본 권한을 제어**하는 설정값(마스크)임.
- 시스템의 최대 기본 권한에서 `umask` 값을 제거하여 최종 기본 권한 결정.
    - 최대 기본 권한:
        - 파일: `666` (`rw-rw-rw-`) (보안상 기본적으로 실행 권한 제외)
        - 디렉토리: `777` (`rwxrwxrwx`)
    - `umask` 값: 제거할 권한을 8진수로 표현 (예: `022`). `0`은 아무것도 제거 안 함, `2`는 쓰기(w) 권한 제거.
- 계산 예시 (`umask`가 `022`인 경우):
    - 새 파일 기본 권한: `666` - `022` = `644` (`rw-r--r--`)
    - 새 디렉토리 기본 권한: `777` - `022` = `755` (`rwxr-xr-x`)
- 확인 및 설정:
    - `umask` 명령어: 현재 설정된 `umask` 값을 8진수로 보여줌.
    - `umask -S` 명령어: 현재 `umask` 설정으로 생성될 파일/디렉토리의 기본 권한을 심볼릭 형태로 보여줌 (예: `u=rwx,g=rx,o=rx`).
    - `umask` 값은 주로 사용자의 셸 시작 파일(예: `~/.bashrc`, `~/.profile`)에서 설정됨.


### **6. 자주 쓰는 명령어**

| 명령어     | 설명              | 예시                                             |
| ------- | --------------- | ---------------------------------------------- |
| chmod   | 파일의 권한을 바꿈      | `chmod 744 file`<br>`chmod u-x,g=r,o-x file`   |
| chown   | 파일의 소유자를 바꿈     | `chown user1 file` <br>`chow user1:user1 file` |
| adduser | 새 유저 추가         | `adduser testuser1`                            |
| passwd  | 추가한 유저의 비밀번호 설정 | `passwd testuser1`                             |
| touch   | 파일 생성           | `touch testfile`                               |
| mkdir   | 디렉토리 생성         | `mk dir testdircory`                           |

# 링크 
## 🔗 아웃링크
```dataview
LIST WITHOUT ID outgoing
FROM ""
WHERE file = this.file
FLATTEN file.outlinks as outgoing
SORT outgoing ASC
```
## 🔙 백링크
```dataview
LIST WITHOUT ID file.link
WHERE contains(file.outlinks, this.file.link)
SORT file.link ASC
```
