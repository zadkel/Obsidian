## **리눅스 패키지 관리 명령어 (yum / dnf)**

리눅스 시스템에서 소프트웨어(패키지)를 쉽게 설치, 업데이트, 삭제하고 의존성을 관리하는 도구에 대한 명령어 모음. Red Hat 계열(CentOS, RHEL, Fedora, Rocky Linux, AlmaLinux 등)에서 주로 사용됨.

- **`yum`:** 오래된 버전에서 사용되던 패키지 관리자.
- **`dnf`:** 최신 버전(Fedora, RHEL 8+, CentOS 8+ 등)의 기본 패키지 관리자. `yum`과 호환되며 명령어 사용법이 거의 동일함. **아래 표의 `yum` 명령어는 대부분 `dnf`로 대체하여 사용 가능함.**

| <div style="width:90px">분류</div> | 명령                              | 설명                                        |
| :------------------------------- | :------------------------------ | :---------------------------------------- |
| 확인                               | `yum help`                      | 도움말 보기                                    |
| 확인                               | `yum repolist`                  | 활성화된 Repository(저장소) 목록 확인                |
| 확인                               | `yum list`                      | 저장소의 모든 패키지 목록 확인 (설치 가능 목록 포함)           |
| 확인                               | `yum list installed`            | 시스템에 현재 설치된 패키지 목록 확인                     |
| 확인                               | `yum list updates`              | 업데이트 가능한 패키지 목록 확인                        |
| 검색                               | `yum search <keyword>`          | 키워드로 패키지 검색                               |
| 확인                               | `yum info <package-name>`       | 특정 패키지의 상세 정보 확인                          |
| 확인                               | `yum deplist <package-name>`    | 특정 패키지의 의존성 관계 확인                         |
| 확인                               | `yum check-update`              | 설치된 패키지 중 업데이트가 있는지 확인 (실제 업데이트는 안 함)     |
| 설치                               | `yum install <package-name>`    | 패키지 설치                                    |
| 삭제                               | `yum remove <package-name>`     | 패키지 삭제                                    |
| 업데이트                             | `yum update [<package-name>]`   | 특정 패키지 또는 전체 시스템 패키지 업데이트 (패키지명 생략 시 전체)  |
| 업데이트                             | `yum upgrade [<package-name>]`  | 패키지 업데이트 (update와 유사하나, 구 버전 처리 방식 약간 다름) |
| 그룹 관리                            | `yum grouplist`                 | 패키지 그룹 목록 확인                              |
| 그룹 관리                            | `yum groupinfo <group-name>`    | 특정 그룹에 포함된 패키지 등 상세 정보 확인                 |
| 그룹 관리                            | `yum groupinstall <group-name>` | 그룹 단위로 패키지 묶음 설치                          |
| 그룹 관리                            | `yum groupremove <group-name>`  | 그룹 단위로 패키지 묶음 삭제                          |
| 캐시 관리                            | `yum clean all`                 | 다운로드된 패키지 파일 및 메타데이터 캐시 삭제 (문제 발생 시 유용)   |


### **자주 사용하는 옵션**

- `-y`: 설치, 삭제, 업데이트 등 진행 시 나타나는 확인 질문(y/N)에 자동으로 'yes'로 응답함. 스크립트 등에서 유용.
    - 예: `yum -y install httpd` (httpd 패키지를 확인 절차 없이 설치)

# 링크 
## 🔗 아웃링크
```dataview
LIST WITHOUT ID outgoing
FROM ""
WHERE file = this.file
FLATTEN file.outlinks as outgoing
SORT outgoing ASC
```
## 🔙 백링크
```dataview
LIST WITHOUT ID file.link
WHERE contains(file.outlinks, this.file.link)
SORT file.link ASC
```
