## **리눅스 기초 (Linux Basics)**

### ### **1. 기본 환경 (Basic Environment)**

- **가상 머신 (VM):** 독립된 리눅스 실행 환경 제공. 시스템 분리 및 테스트에 유용함.
- **리눅스 배포판 (Distribution):** 리눅스 커널 + 각종 도구/프로그램 집합. (예: Ubuntu, CentOS, Rocky Linux 등) 목적에 맞게 선택.
- **터미널 에뮬레이터 (Terminal):** 명령어 기반 인터페이스(셸)에 접근하는 창. 리눅스 제어의 기본 통로.

### **2. 시스템 핵심 요소 (Core System Components)**

- **커널 (Kernel):** 리눅스 운영체제의 심장. 하드웨어 제어 및 시스템 자원 관리.
- **셸 (Shell):** 명령어 해석기. 사용자와 커널 간의 다리 역할 (예: Bash).
- **파일 시스템 (Filesystem):** 데이터를 저장하고 체계적으로 관리하는 계층적 구조.

### **3. 주요 개념 및 관리 (Key Concepts & Management)**

- **사용자 (User):**
    - `root`: 시스템 관리자 계정 (모든 권한).
    - 일반 사용자: 제한된 권한을 가진 계정.
- **권한 관리 (Privilege Management):**
    - `su`: 다른 사용자로 전환 (주로 `su -` 로 root 전환).
    - `sudo`: 일반 사용자가 특정 명령어를 임시로 root 권한으로 실행. (권장 방식)
- **파일 시스템 구조 (FHS - Filesystem Hierarchy Standard):** 표준화된 디렉토리 레이아웃.
    - `/etc`: 시스템 설정 파일 위치.
    - `/home`: 일반 사용자 홈 디렉토리 위치.
    - `/var`: 로그, 메일 등 변경되는 데이터 저장 위치.
    - 나머지는 [[리눅스 파일 시스템 구조]] 참조
- **기본 명령어 (Basic Commands):** 파일/디렉토리 조작, 시스템 정보 확인 등
    - [[리눅스 기본 명령어]] 참조
- **패키지 관리 (Package Management):** 소프트웨어 설치, 업데이트, 제거 자동화 도구.
    - Red Hat 계열: `yum`, `dnf`
    - Debian 계열: `apt`
    - [[리눅스 패키지 명령어]] 참조
- **서비스(데몬) 관리 (Service Management):** 백그라운드에서 동작하는 프로그램(서비스) 제어.
    - `systemd` (`systemctl` 명령어 사용): 서비스 시작, 중지, 재시작, 상태 확인, 부팅 시 자동 실행 설정 등.
    - [[리눅스 서비스 관리 명령어]]
- **방화벽 (Firewall):** 네트워크 트래픽을 규칙에 따라 허용/차단하여 시스템 보안 강화.
    - [[리눅스 방화벽]] 참조

## 🔗 아웃링크
```dataview
LIST WITHOUT ID outgoing
FROM ""
WHERE file = this.file
FLATTEN file.outlinks as outgoing
SORT outgoing ASC
```
## 🔙 백링크
```dataview
LIST WITHOUT ID file.link
WHERE contains(file.outlinks, this.file.link)
SORT file.link ASC
```
