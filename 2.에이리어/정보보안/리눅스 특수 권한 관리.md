## **리눅스 특수 권한 (SetUID, SetGID, Sticky Bit)**

### **1. 개요 (Overview)**

- 리눅스 파일/디렉토리에는 기본적인 읽기(r), 쓰기(w), 실행(x) 권한 외에 추가적인 동작을 제어하는 **특수 권한**이 존재함.
- 대표적으로 SetUID, SetGID, Sticky Bit가 있으며, `chmod` 명령어를 통해 설정 가능함.
- 이 권한들은 주로 시스템 보안 및 협업 환경에서의 파일 관리를 위해 사용됨.

### **2. SetUID (Set User ID)**

- **목적:** 실행 파일(`x` 권한 필요)에 설정 시, 해당 파일을 **실행하는 사용자의 권한이 아닌 파일 소유자(Owner)의 권한으로** 프로세스를 실행시킴.
- **주요 용도:** 일반 사용자가 자신의 정보 변경 등을 위해 잠시 루트(root) 권한이 필요한 명령어 실행 시 (예: `passwd` - `/etc/shadow` 파일 수정 권한 필요).
- **보안:** 강력한 기능이므로 보안상 신중하게 사용해야 함. SetUID가 설정된 프로그램(특히 root 소유)에 취약점이 있을 경우, 시스템 권한 탈취로 이어질 수 있음.
- **표시:**
    - `ls -l`: 소유자(Owner)의 실행 권한 자리에 `s`로 표시됨 (`rwsr-xr-x`). 소유자 실행 권한(`x`)이 없으면 대문자 `S`로 표시됨 (`rwSr-xr-x`).
    - `chmod` (8진수): 숫자 `4000`에 해당. 예: `chmod 4755 <파일명>` (`rwsr-xr-x`).
    - `chmod` (심볼릭): `u+s`. 예: `chmod u+s <파일명>`.

### **3. SetGID (Set Group ID)**

- **목적 (실행 파일):** 실행 파일(`x` 권한 필요)에 설정 시, 해당 파일을 실행하는 프로세스가 **파일의 그룹(Group) 권한으로** 실행됨. SetUID와 유사하나 그룹 권한 기준.
- **목적 (디렉토리):** **더 흔하게 사용됨.** 디렉토리에 설정 시, 해당 디렉토리 내부에 **새로 생성되는 파일이나 하위 디렉토리의 소유 그룹이 상위 디렉토리의 그룹으로 자동 설정됨** (파일 생성 사용자의 기본 그룹 무시).
- **주요 용도 (디렉토리):** 여러 사용자가 공동 작업하는 프로젝트 디렉토리에서 생성되는 파일들의 그룹 소유권을 일관되게 유지하여 권한 관리를 용이하게 함.
- **표시:**
    - `ls -l`: 그룹(Group)의 실행 권한 자리에 `s`로 표시됨 (`rwxrwsr-x`). 그룹 실행 권한(`x`)이 없으면 대문자 `S`로 표시됨 (`rwxrwSr-x`).
    - `chmod` (8진수): 숫자 `2000`에 해당. 예: `chmod 2775 <디렉토리명>` (`rwxrwsr-x`).
    - `chmod` (심볼릭): `g+s`. 예: `chmod g+s <파일 또는 디렉토리명>`.

### **4. Sticky Bit**

- **목적:** 주로 **디렉토리**에 설정됨. 해당 디렉토리에 **쓰기(w) 권한이 있는 모든 사용자**가 파일을 생성할 수는 있지만, **파일 삭제/이름 변경은 해당 파일의 소유자, 디렉토리의 소유자, 또는 root 사용자만 가능하도록 제한**함.
- **주요 용도:** 여러 사용자가 파일을 자유롭게 생성하지만, 다른 사용자의 파일을 함부로 삭제/수정할 수 없어야 하는 공용 디렉토리 (예: `/tmp`).
- **표시:**
    - `ls -l`: 기타 사용자(Others)의 실행 권한 자리에 `t`로 표시됨 (`rwxrwxrwt`). 기타 사용자 실행 권한(`x`)이 없으면 대문자 `T`로 표시됨 (`rwxrwxrwT`).
    - `chmod` (8진수): 숫자 `1000`에 해당. 예: `chmod 1777 /tmp` (`rwxrwxrwt`).
    - `chmod` (심볼릭): `+t`. 예: `chmod +t <디렉토리명>`.

# 링크 
## 🔗 아웃링크
```dataview
LIST WITHOUT ID outgoing
FROM ""
WHERE file = this.file
FLATTEN file.outlinks as outgoing
SORT outgoing ASC
```
## 🔙 백링크
```dataview
LIST WITHOUT ID file.link
WHERE contains(file.outlinks, this.file.link)
SORT file.link ASC
```
