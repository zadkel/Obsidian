## 1. VPC (Virtual Private Cloud)

- **개념:** 사용자의 AWS 계정 전용으로 **논리적으로 격리된 가상의 네트워크 환경**을 의미함. 사용자는 이 가상 네트워크 공간을 직접 정의하고 제어할 수 있음.
- **IP 범위:** VPC 생성 시, CIDR(사이더) 블록 표기법을 사용하여 사설 IPv4 주소 범위를 지정함 (예: `10.0.0.0/16`). AWS는 `/16`부터 `/28`까지의 넷마스크를 지원함.

## 2. VPC 특징 및 기능

- **완전한 네트워크 제어:** IP 주소 범위, 서브넷 생성, 라우팅 테이블 구성, 네트워크 게이트웨이 설정 등 네트워크 환경의 거의 모든 측면을 사용자가 직접 제어 가능함.
- **논리적 격리:** 다른 가상 네트워크와 완전히 분리되어 있어 보안성을 높임.
- **맞춤형 환경 구성:** 온프레미스 네트워크 환경과 유사하게 AWS 클라우드 내에 네트워크 토폴로지 구성 가능.

## 3. 주요 구성 요소

- **IP 주소 범위 (CIDR):** VPC 전체에서 사용할 사설 IP 주소 대역. (예: `10.0.0.0/16`)
- **서브넷 (Subnet):**
    - VPC의 IP 주소 범위를 더 작게 나눈 네트워크 영역.
    - 각 서브넷은 특정 **가용 영역(Availability Zone - AZ)** 내에 존재해야 하며, AZ 간 장애 격리 제공.
    - 인터넷 연결 여부에 따라 **퍼블릭 서브넷**(Internet Gateway 통해 외부 통신 가능)과 **프라이빗 서브넷**(외부 직접 통신 불가)으로 구분 가능.
- **라우팅 테이블 (Routing Table):**
    - 네트워크 트래픽이 어디로 가야 하는지를 결정하는 **경로 규칙(Route)들의 모음**.
    - 각 서브넷은 반드시 하나의 라우팅 테이블과 연결되어, 해당 서브넷에서 발생하는 트래픽의 전달 경로를 정의함.
- **게이트웨이 (Gateway):**
    - VPC가 다른 네트워크(인터넷, 온프레미스 네트워크 등)와 통신할 수 있도록 하는 연결 지점.
    - **인터넷 게이트웨이 (IGW):** VPC와 인터넷 간의 통신 활성화. 퍼블릭 서브넷에서 사용.
    - **NAT 게이트웨이 / NAT 인스턴스:** 프라이빗 서브넷의 인스턴스가 인터넷으로 아웃바운드 통신은 가능하게 하되, 외부에서 직접 인스턴스로 접근(인바운드)은 차단.
    - **가상 프라이빗 게이트웨이 (VPG):** 온프레미스 네트워크와 VPN 연결 시 사용.
- **엔드포인트 (Endpoint):**
	- VPC 내 리소스(예: EC2 인스턴스)가 인터넷 게이트웨이, NAT 디바이스, VPN 연결 등 없이 **AWS 네트워크 내부에서 비공개적으로** 지원되는 AWS 서비스(예: S3, DynamoDB) 또는 VPC 엔드포인트 서비스(PrivateLink 기반)에 **연결**할 수 있도록 지원하는 기능임.
	- 쉽게 말해, 인터넷 없이 내부 자원에 접근 가능.
	- 종류: 
		- 인터페이스 엔드포인트(ENI 생성, 다수 서비스 지원)
		- 게이트웨이 엔드포인트(라우팅 테이블 경로 지정, S3/DynamoDB 지원).
	- 이점:
		- 보안 강화 (트래픽이 퍼블릭 인터넷으로 나가지 않음)
		- 안정적인 연결성 제공.


## 4. 네트워크 보안

- **보안 그룹 (Security Group):**
    - 인스턴스 수준에서 적용되는 **상태 기반(Stateful) 가상 방화벽**. 인스턴스로 들어오고 나가는 트래픽(Inbound/Outbound) 규칙 제어.
- **네트워크 ACL (Network Access Control List - NACL):**
    - 서브넷 수준에서 적용되는 **상태 비저장(Stateless) 방화벽**. 서브넷으로 들어오고 나가는 트래픽 규칙 제어. 보안 그룹보다 앞단에서 동작.

## 5. 연결 옵션

- VPC를 온프레미스 데이터 센터나 다른 VPC와 연결하는 다양한 방법 제공.
- **AWS Site-to-Site VPN:** 인터넷을 통해 암호화된 VPN 터널을 생성하여 VPC와 온프레미스 네트워크 연결. 가상 프라이빗 게이트웨이(VPG) 필요.
- **AWS Direct Connect:** AWS와 온프레미스 데이터 센터 간의 전용 프라이빗 네트워크 연결 제공 (인터넷 경유 안 함). 안정적이고 높은 대역폭 보장.
- **VPC 피어링 (Peering):** 두 개의 VPC 간에 비공개적으로 트래픽을 라우팅할 수 있도록 연결.

## 6. IP 주소 할당

- **사설 IP (Private IP):** VPC 내의 모든 EC2 인스턴스는 시작 시 해당 서브넷의 IP 범위에서 자동으로 사설 IP 주소를 할당받음. VPC 내부 통신에 사용됨.
- **공인 IP (Public IP) / 탄력적 IP (Elastic IP):**
    - 인터넷과의 통신이 필요한 인스턴스에는 공인 IP 주소를 할당할 수 있음.
    - **Elastic IP (EIP):** AWS 계정에 할당되는 **고정된 공인 IP 주소**. 인스턴스를 중지/시작해도 IP 주소가 변경되지 않아야 할 때 유용하게 사용됨. EIP는 EC2 인스턴스 등에 연결하여 사용.

## 🔗 아웃링크

```dataview
LIST WITHOUT ID outgoing
FROM ""
WHERE file = this.file
FLATTEN file.outlinks as outgoing
WHERE endswith(lower(outgoing.path), ".md")
SORT outgoing ASC
```
